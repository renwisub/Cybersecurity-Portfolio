 Network Fundamentals: Diagnostics, Packet Capture & TCP Handshake Analysis

## Задача

Освоить базовые сетевые инструменты диагностики (ipconfig, ping, tracert, nslookup) и захватить
реальный сетевой трафик в Wireshark, чтобы понять, как выглядит DNS-резолвинг и TCP-соединение
на уровне пакетов — фундамент, без которого невозможно читать сетевые логи в SOC.

## Что искал

- Конфигурацию собственного сетевого интерфейса и маршрут пакетов до внешнего хоста
- DNS-запрос и DNS-ответ в живом трафике (протокол, порт, TTL)
- Полный TCP 3-way handshake (SYN → SYN-ACK → ACK) с разбором флагов и номеров пакетов

## Процесс

**1. Диагностика сети**

```
ipconfig /all      → IP: 192.168.31.152, Gateway: 192.168.31.1, DNS: 192.168.31.1
ping 8.8.8.8        → 4/4 пакета, TTL=106, среднее время 31ms
tracert google.com  → 24 хопа до 64.233.165.101
nslookup google.com → преобразование домена в набор IPv4/IPv6 адресов
```

**2. Захват DNS-трафика в Wireshark**

Захватил живой DNS-запрос к `api.trancy.org`:
- Source: `192.168.31.152` → Destination: `192.168.31.1` (локальный DNS-сервер)
- Query type: A (IPv4)
- Полученный IP: `8.47.69.0`, TTL записи: `17`

**3. Разбор TCP 3-way handshake**

Захватил полное установление TCP-соединения к `108.177.14.100:443`:

| Шаг | Flags | Seq | Ack | Смысл |
|---|---|---|---|---|
| 1 | SYN | 0 | — | Клиент инициирует соединение |
| 2 | SYN, ACK | 0 (сервер) | 1 | Сервер подтверждает и готов к соединению |
| 3 | ACK | — | 1 | Клиент подтверждает, соединение установлено |

Source port клиента: `64963`, Destination port: `443` (HTTPS)

## Вывод

Понял, как ОС резолвит домены через DNS (UDP/53), как устанавливается надёжное TCP-соединение
через 3-way handshake, и что каждый пакет несёт уникальные Sequence/Acknowledgment номера для
гарантии целостности передачи. Это база для дальнейшего чтения firewall-логов (Event 5156) и
анализа сетевых индикаторов компрометации.

**Открытые вопросы для дальнейшего изучения:** почему TTL уменьшается на каждом хопе и как это
используется для traceroute; как ОС распределяет порты между приложениями.

## Инструменты

Windows CLI (`ipconfig`, `ping`, `tracert`, `nslookup`) · Wireshark
